========================================================================================================================
SAMENVATTING STRUCTURELE VERSCHILLEN
========================================================================================================================

ORIGINELE JSON:
  - Character: volledig profiel (name, role, bio, music)
  - Intro: text + background + parallaxEffect + music reference
  - Questions: location, background, parallaxEffect + scoring fields
  - Tradeoffs: structured dict {title, text, payoffParallaxEffect}
  - GEBRUIK: Game UI met visuele effecten en scoring

DOCUMENT REFERENCE FILES:
  - Character: basis info (name, role, description, bio)
  - Intro: alleen tekst
  - Questions: alleen basis content (context, text, options, tradeoffs)
  - Tradeoffs: simplified string values
  - GEBRUIK: Clean content reference, geen UI metadata
  
VOORDEEL DOCUMENT REFERENCE:
  - Lesbaarder voor inhoudelijke vergelijking
  - Geen game-specifieke metadata
  - Makkelijker om inhoud te updaten
  - Kan als template dienen voor andere implementaties
